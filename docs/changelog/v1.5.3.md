# nerve-wml v1.5.3 — methodology cross-checks

**Release date:** 2026-04-21
**Scope:** methodology module + paper Test (7). No change to the
v1.2.3 scientific baseline or to any of the Claim A / Claim B
empirical measurements. Fully backward-compatible.

## Why

v1.5.2 shipped with an explicit caveat in the README Cross-lab
methodology commitment section: the MI/H = 0.91–0.96 headline of
Claim B was measured with a single estimator (argmax one-hot +
entropy-normalised MI), and a `bouba_sens` v0.5.0-style three-check
robustness audit (null-model, bootstrap CI, multi-estimator) was
flagged as pre-submission work.

v1.5.3 delivers all three checks, with the fourth estimator (MINE)
that `bouba_sens` section 6.3 requested for completeness.

## What's new

### New submodule: `nerve_wml.methodology`

Five primitives, pure numpy/torch/scipy, all importable directly
and designed to be consumed by `bouba_sens` via
`from nerve_wml.methodology import null_model_mi, ...`.

| Primitive | Purpose |
|---|---|
| `null_model_mi` | Permutation significance test for MI |
| `bootstrap_ci_mi` | Non-parametric CI via resampling |
| `mi_plugin_discrete` | Baseline entropy-normalised plug-in |
| `mi_miller_madow_discrete` | Bias-corrected plug-in |
| `mi_kraskov_ksg_continuous` | KSG k-NN on continuous embeddings |
| `mi_mine` | MINE DV neural variational bound |
| `entropy_discrete` | Plug-in Shannon entropy helper |

Full API and rationale in docstrings; 34 unit tests cover
identical-codes, independent-codes, shape validation, reproducibility,
KSG monotonicity, and MINE recovery on synthetic gaussians.

### New paper Test (7) — Multi-estimator robustness

Adds `papers/paper1/main.tex` section Information Transmission
Test (7) with Table 3 reporting all four MI estimators on the
same 3-seed `run_w2_hard` pipeline:

| Estimator | Level | Mean |
|---|---|---|
| Plug-in MI/H(a) | post-VQ | 0.932 |
| Miller-Madow MI/H(a) | post-VQ | 0.939 |
| Kraskov KSG / H(a) | pre-VQ | 0.092 |
| MINE / H(a) | pre-VQ | 0.985 |

**Finding 1 (discrete cross-estimator robustness).** Plug-in and
Miller-Madow agree within Δ = 0.007 ≤ 0.10. The post-VQ MI/H
headline is robust to bias correction — the `bouba_sens`
multi-estimator check passes on the discrete side.

**Finding 2 (continuous-estimator divergence).** KSG and MINE
diverge by an order of magnitude on the pre-VQ continuous
embeddings. Two interpretations compete:

- **(a) KSG underestimates** — curse of dimensionality at d=16,
  N=2000 biases kNN density low; true pre-VQ MI is close to 0.99.
  Under (a), pre- and post-VQ MI are comparable, no VQ
  amplification.
- **(b) MINE overestimates** — DV-bound variance + finite-sample
  critic overfitting inflates the estimate; true pre-VQ MI is
  closer to 0.09. Under (b), VQ discretisation amplifies the
  shared-information fraction by ~10× (concentrates agreement
  onto the 7–27 of 64 effectively-used codes).

Huh 2024 mutual-kNN (Test (6)) independently reports 0.174 at
k=10, aligning with KSG (factor ~2). Two out of three continuous
methods favour interpretation (b) but this is not a rigorous
arbitration; we report both figures and flag the ambiguity.

The discrete-side Δ = 0.007 is independent of the continuous
divergence and remains the load-bearing robustness claim.

### N-3 gate investigation closure (backfilled from earlier commits)

Three ablations all yield Δ = 0 on correct pipelines, confirming
the gamma/theta invariant as a formal correctness contract rather
than a differential computational mechanism. Full reasoning in
`docs/research-notes/n3-gate-role.md`. Paper Limitations section
extended in the v1.5.2 cycle (commit `a866b28`) already reflects
this finding.

## Reproducibility

- `tests/golden/codes_mlp_lif.npz` (2.1 MB) — 3-seed MLP+LIF
  argmax codes plus pre-VQ continuous embeddings; all downstream
  measurement scripts consume this artefact without retraining.
- `papers/paper1/figures/mi_null_model.json`, `mi_bootstrap_ci.json`,
  `mi_multi_estimator.json`, `mi_mine.json` — primary results.
- Regeneration: produce NPZ via
  `uv run python scripts/save_codes_for_checks.py`, then run any
  `measure_*` script in `scripts/`.
