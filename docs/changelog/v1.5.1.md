# nerve-wml v1.5.1 — version metadata sync

**Release date:** 2026-04-21
**Scope:** packaging patch.

## Fixed

- `pyproject.toml` `[project].version` bumped from `1.4.0` to `1.5.1` so
  that wheels built from the v1.5.x line report the correct version in
  `pip list` / `python -c "import nerve_wml; print(nerve_wml.__version__)"`.
  v1.5.0 shipped with `version = "1.4.0"` still in `pyproject.toml` — no
  functional impact, but a stale metadata field that confused PyPI-style
  consumers.

## Unchanged

- All runtime behaviour. The 102 multiplexer / transducer / spectrogram /
  dream-bridge tests all pass on v1.5.0 and v1.5.1 identically.

## CITATION.cff policy

Per-release version DOIs are no longer hard-pinned in `CITATION.cff` —
only the **concept DOI** `10.5281/zenodo.19656342` stays, resolving to
whatever the latest Zenodo record is. This avoids the n-per-release
`CITATION.cff` churn. Reviewers who want to cite a specific version pin
the version-DOI from the Zenodo record page (or from GitHub release
notes) directly.
